# Deployment Guide 🚀

Synchro is designed to be easily deployable on Vercel with a database-less architecture.

## 1. Google Cloud Console Setup

Since Synchro uses Google OAuth for authentication and ownership verification, you must set up a project in the [Google Cloud Console](https://console.cloud.google.com/):

1. **Create a Project**: Give it a name like "Synchro."
2. **OAuth Consent Screen**:
   - User Type: External.
   - Add scopes: `.../auth/userinfo.email`, `.../auth/calendar.events` (optional for history sync).
3. **Credentials**:
   - Create an **OAuth 2.0 Client ID**.
   - Application Type: Web Application.
   - **Authorized JavaScript Origins**: 
     - `http://localhost:3000`
     - `https://your-domain.vercel.app`
   - **Authorized Redirect URIs**: 
     - `http://localhost:3000`
     - `https://your-domain.vercel.app` (Note: Google requires an exact match. Ensure there is no trailing slash).

## 2. Environment Variables

Create a `.env.local` file (for local development) or add these to your Vercel project settings:

```bash
# REQUIRED
NEXT_PUBLIC_GOOGLE_CLIENT_ID=your-google-client-id.apps.googleusercontent.com

# OPTIONAL (for personalized analytics if needed)
NEXT_PUBLIC_SITE_URL=https://your-domain.vercel.app
```

**Note:** No database URL is required. Synchro uses in-memory signaling and client-side storage.

## 3. Deploy to Vercel

1. **Push to GitHub**: Push your code to a repository.
2. **Connect to Vercel**: Import the repository into Vercel.
3. **Configure Environment Variables**: Add the variables listed above.
4. **Deploy**: Vercel will build and deploy the application.

## 4. Stability Check

Ensure your "Authorized Redirect URIs" in Google Cloud Console exactly match the domain generated by Vercel. If you see a `redirect_uri_mismatch` error, update the Google Console with the exact URL provided in the error message.
